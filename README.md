# Приложение «Трекер задач» (Kanban-доска)

REST API бэкенд для управления задачами. Система поддерживает создание иерархических задач, автоматический расчёт их 
временных параметров, бесконфликтное планирование расписания, отслеживание истории просмотров и сохранение данных в 
файлы.

---

## Архитектура и структура данных

В проекте выделены три типа задач (перечисление `TypeOfTasks`), наследуемые от базового класса `Task`:
- **Task (Простая задача)** — задача, содержащая поля: начало выполнения - `startTime` (`LocalDateTime`) и 
продолжительность `duration` (`Duration`), а также вычисляемое поле окончания выполнения задачи `getEndTime()`.
- **Epic (Эпик)** — крупная цель, агрегирующая внутри себя список идентификаторов подзадач `subtasksIds`.  Статус и 
- временные рамки эпика рассчитываются автоматически на основе его содержимого.
- **Subtask (Подзадача)** — составная часть эпика, жестко связанная с ним через поле `epicId`.

### Жизненный цикл и статусы (TaskStatus):
Задачи проходят через статусы: `NEW` ➡️ `IN_PROGRESS` ➡️ `DONE`.

---

## Ключевые компоненты и алгоритмы

1. **Менеджеры задач (TaskManager)**
    - **InMemoryTaskManager** — хранит задачи в оперативной памяти (`HashMap`) и управляет расписанием.
    - **FileBackedTaskManager** — автоматически сохраняет (`save()`) состояние менеджера в CSV-файл при любых изменениях
   и восстанавливает данные (`loadFromFile`) при перезапуске приложения.

2. **Валидация расписания (Пересечения по времени)**
    - Все задачи с установленным временем начала автоматически сортируются в коллекции `TreeSet` (`prioritizedSet`).
    - Перед добавлением или обновлением задачи система проверяет её на наложение временных интервалов с другими 
   задачами. В случае конфликта выбрасывается исключение `TimeOverlapException`.

3. **Оптимизированный менеджер истории (HistoryManager)**
    - Реализован в классе `InMemoryHistoryManager` на стыке **кастомного двусвязного списка (`Node<T>`)** и 
   **`HashMap`**.
    - Позволяет добавлять просмотры задач и удалять их за константное время **O(1)** без линейного перебора элементов.
    - Автоматически перемещает повторно просмотренные задачи в конец списка, исключая дубликаты в истории.

4. **Сетевой слой (HttpTaskServer)**
    - Запускает REST API сервер на базе встроенного Java `HttpServer` (порт `8080`).
    - Синхронизирует данные через JSON с помощью библиотеки `Gson` и кастомных адаптеров для типов данных `Duration` и
   `LocalDateTime`.

---

## Спецификация REST API (Эндпоинты)

Система поддерживает стандартные HTTP-методы (`GET`, `POST`, `DELETE`) для следующих ресурсов:

- `/tasks` — Управление простыми задачами (все или конкретная по `?id=`).
- `/epics` — Управление эпиками и просмотр их содержимого.
- `/subtasks` — Управление подзадачами.
- `/history` — Получение актуального списка истории просмотров без дубликатов.
- `/prioritized` — Получение списка всех задач, упорядоченных по времени начала выполнения.

## Технологический стек

*   Язык и среда:  Java 21 (Amazon Corretto).
*   Сетевой слой: Встроенный Java `com.sun.net.httpserver.HttpServer` для реализации REST API.
*   Сериализация и работа с JSON: Google Gson 2.9.0 (с кастомными адаптерами для типов `Duration` и `LocalDateTime`).
*   Тестирование:** JUnit 5.8.1 (Unit-тестирование логики менеджеров задач, истории просмотров и валидации временных пересечений).
*   Хранение данных: Внутренние структуры данных (`HashMap`, `TreeSet`), кастомный двусвязный список, а также персистентный файловый бэкенд на базе CSV (`FileBackedTaskManager`).

## Системные требования

* Операционная система: Windows, macOS, Linux

## Инструкция по развертыванию и запуску в IntelliJ IDEA

1. Клонируйте репозиторий с проектом:
   git clone https://github.com

2. Откройте проект в IntelliJ IDEA:
   Файл -> Открыть -> Выберите корневую папку проекта.

3. Настройте версию Java (JDK 21):
   * Перейдите в Файл -> Структура проекта (Project Structure).
   * Вкладка Проект (Project) -> Установите SDK: Amazon Corretto 21.

4. Запустите HTTP-сервер:
   * Найдите класс HttpTaskServer (в папке src).
   * Нажмите на зеленую стрелку (Run) рядом с методом main.
   После запуска сервер начнет принимать REST-запросы по адресу http://localhost:8080.

5. Запуск тестов:
   * Нажмите правой кнопкой мыши на папку test.
   * Выберите Запустить все тесты (Run 'All Tests'). Все JUnit 5 тесты выполнятся автоматически.
   
---

## Альтернативный запуск через консоль

Если вы запускаете проект без IDE, скомпилируйте файлы вручную с подключением необходимых библиотек:

1. Скомпилируйте исходный код и тесты (с указанием пути к библиотекам gson и junit в вашем репозитории):
   javac -d out -cp "lib/*" src/**/*.java test/**/*.java

2. Запустите сервер:
   java -cp "out:lib/*" HttpTaskServer
